When you do need the Python ecosystem.

Pulse proved Jac stands on its own. The Metrics Workbench is the other half of the story: a real analytics app where pandas, pyod and statsmodels do the numeric heavy lifting JavaScript has no answer for — imported straight into the Jac source, behind the same walkers that power the CLI, the REST API, and this dashboard.

pandas dataframes pyod IForest statsmodels SARIMAX by llm insight + suggestions + agent 129,600 rows

Python interop — the load-bearing kind

The numeric work lives in one Python module (analytics.py) and is imported directly into the Jac source. This is the escape-hatch flavour of interop: reach for the mature scientific stack when it earns its keep, without leaving your app.

# api.jac — the graph model + walkers are Jac; the math is Python, imported in
import from analytics {
    describe_dataset  as py_describe,      # pandas — ingest 129k rows
    detect_anomalies  as py_detect,        # pyod IForest
    forecast          as py_forecast,      # statsmodels SARIMAX + conf. intervals
    web_dashboard     as py_web_dashboard  # pandas + both, chart-ready
}

SARIMAX forecast

A seasonal ARIMA fit projects 30 days ahead with a 95% confidence band that widens with the horizon — the teal cone on the chart.

Why this is the capstone. Pulse writes its z-score in pure Jac because it can. The workbench doesn't pretend to reimplement SARIMAX or IForest in the browser — it imports the real thing. Same graph, same walkers, same by llm; heavier engine.

AI, three ways — all typed

The LLM's inputs and outputs are Jac types, not prompt strings. Three by llm functions power three different features:

# the signatures ARE the prompts — structured in, structured out
def narrate(kind: str, metric: str, findings_json: str) -> InsightPayload by llm();
def suggest_metrics(dataset: str, columns_json: str, sample_json: str) -> list[MetricSuggestion] by llm();
def decide_next(question: str, metric: str, recent: str) -> Direction by llm();  # an enum
by llm · narrate

Explains the anomalies

Given the run's numbers, the LLM writes a typed Insight — "highest anomaly score 0.818 on May 2 with revenue $51.16M".

by llm · suggest_metrics

Proposes new metrics

Given the columns + a sample, it returns 10 candidate metrics with pandas/SQL expressions and rationales — "Refund Rate = SUM(refunds)/SUM(gross_revenue)".

The third — decide_next — returns a Jac enum that steers a walker through the graph. This is an agent as an object-spatial traversal: the LLM decides where to hop next at each node it visits.

walker investigate {
    can at_metric with Metric entry {
        decision = decide_next(self.question, here.name, recent_summary);
        if decision == Direction.LOOK_AT_RUNS { visit [recent[-1]]; }
        elif decision == Direction.HOP_SIBLING { visit [sibling_metric]; }
        else { self._finalize(); }   # narrate what it found
    }
}
